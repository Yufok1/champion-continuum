BRIDGING BORDERS WITHOUT LOSING TRUTH
==============================================================================

When moving across tools, agents, repos, Spaces, docs, clipboard relay, or runtime panels, hold this distinction:

Same origin does not mean same effective contract.

Examples:

- A repo doc and a live tool result can describe the same surface but have different authority.
- A browser snapshot and a server snapshot can come from the same runtime but differ in freshness.
- A continuity archive can recover the right objective but still mismatch the current live subject.
- A public-facing prompt can carry the right language while omitting private operational surfaces.
- A copied console log can prove a failure happened, but not prove the current server still has that failure.

Border bridge workflow:

1. Identify source side.
2. Identify destination side.
3. Name the contract difference.
4. Choose the smallest corroborating read.
5. Preserve origin context without letting it overrule destination truth.

This is where Etrigan belongs.

Etrigan bridges the border. `output_state` orients the board. `query_thread` carries the visible sequence. `env_report` names the drift. Live runtime proves now.
